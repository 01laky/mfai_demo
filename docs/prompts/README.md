# AI prompts

**Retention:** These prompt files are **kept for the long term** — do **not** delete them to “clean up”; they are the reusable specs for future agent runs and reviews.

Specs meant to be **copied into an AI agent chat** to implement features without losing context. They are not end-user documentation.

### Checklist conventions (`[ ]` vs `[x]`)

| Kind                    | Where                                                                                                                                                                                                        | What to do in **`many_faces_main`** git                                                                                                                                                                                     |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **PR / agent template** | Most prompts: unchecked `[ ]` lists (e.g. **§9–§10**, **§15** in `unit-test-gap-fill-…`, deliverables in **security-hardening**, **monorepo-dependency-audit**, **eslint10-…**, \*\*react-hooks-compiler-…`) | **Leave `[ ]` in the file.** Copy the section into the PR or issue and tick there. Do not mass-replace with `[x]` in the canonical prompt unless you are recording a **fully completed** one-off engagement (see next row) — **[`smtp-mailer-java-grpc-worker-agent-prompt.md`](./smtp-mailer-java-grpc-worker-agent-prompt.md)** is one such completed record (**§11–§12**). |
| **Completed run log**   | `mermaid-documentation-diagrams-agent-prompt.md` (many `[x]` + “Status: Done” / reset instructions)                                                                                                          | The `[x]` marks document a **finished** diagram pass. If you schedule a **full re-audit from scratch**, reset the listed lines back to `[ ]` per that file’s instructions.                                                  |
